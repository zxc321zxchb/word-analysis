## 文档分页查询接口设计

### 1. 添加新的API模型
- 在 `src/doc_analysis/api/models.py` 中添加：
  - `DocumentBriefResponse`：包含文档基础信息（id、原始文件名、文件大小、创建时间等）
  - `DocumentListResponse`：分页响应模型，包含文档列表和分页信息

### 2. 扩展CRUD功能
- 在 `src/doc_analysis/db/crud.py` 中添加：
  - `get_documents_with_pagination()` 函数：支持分页查询文档列表
  - 实现基于 `offset` 和 `limit` 的分页逻辑

### 3. 添加API路由
- 在 `src/doc_analysis/api/routes.py` 中添加：
  - `GET /documents` 接口：支持分页查询文档列表
  - 接受 `page` 和 `page_size` 查询参数
  - 返回文档基础信息列表，不包含完整的章节内容

### 4. 功能特点
- 分页查询减轻前端负担，避免一次性加载大量数据
- 返回的基础信息包含 `document_id`，可用于后续查询详细信息
- 支持按创建时间排序，最新的文档优先显示
- 保持与现有API风格一致，使用相同的响应格式

### 5. 使用方式
```
# 获取文档列表（第1页，每页10条）
GET /api/v1/documents?page=1&page_size=10

# 响应示例
{
  "items": [
    {
      "id": 1,
      "original_filename": "项目需求文档.docx",
      "file_size": 1024000,
      "created_at": "2024-01-28T10:00:00",
      "sections_count": 100
    }
  ],
  "total": 42,
  "page": 1,
  "page_size": 10,
  "pages": 5
}

# 然后使用document_id查询详细信息
GET /api/v1/documents/1
```
